What we are testing

This prototype will be used to test and end-to-end data entry scenario. This scenario will require the user to make use of CCaRD, CRDS, R&S and Adjustments.

In this round, we aim to understand:

  • How do users feel about being able to input data and calculate dates in one system? (ie not switching between NOMIS and CRDS)
  • Do users pause or stop at any point? If so, why? (eg to check info?) or do in one go?
  • Do users ever want to go back and review info? (eg before moving to next product)?
